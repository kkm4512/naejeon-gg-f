# 내전.gg 기획서

> 리그 오브 레전드 내전 전적 검색 사이트  
> 프로젝트명: `naejeon-gg`  
> 프레임워크: Nuxt 3 (SSR)  
> DB: Firebase Firestore (기존 Discord 봇과 공유)

---

## 1. 프로젝트 개요

Discord 봇(`naejeonhajadiscordbot`)이 수집한 내전 게임 데이터를 웹에서 시각적으로 탐색할 수 있는 전적 검색 사이트. op.gg 스타일의 UX를 목표로 하며, Nuxt SSR을 활용해 초기 로딩 성능과 SEO를 확보한다.

---

## 2. 기술 스택

| 항목 | 선택 |
|------|------|
| 프레임워크 | Nuxt 3 |
| 렌더링 | SSR (Server-Side Rendering) |
| 스타일 | Tailwind CSS |
| DB | Firebase Firestore (firebase-admin, server-side only) |
| 챔피언 이미지 | Riot Data Dragon CDN |
| 배포 | Vercel (Node 서버 모드) |

---

## 3. Firestore 데이터 구조 (기존)

```
reports/{gameId}
  ├── overview
  │   ├── gameMode          // "CLASSIC" | "KIWI"
  │   ├── gameType          // "CUSTOM_GAME"
  │   ├── gameDuration      // 초 단위
  │   ├── gameCreationDate  // timestamp
  │   └── teamBans          // { "100": [...], "200": [...] }
  └── stats[]
      ├── summoner          // 소환사 이름
      ├── championId        // 챔피언 ID
      ├── kills / deaths / assists
      ├── win               // boolean
      ├── teamId            // 100 | 200
      ├── totalDamageDealtToChampions
      └── goldEarned
```

## 4. 페이지 구성

### 4-1. 메인 페이지 `/`

- 사이트 소개 + 소환사 검색창
- **누적 랭킹 카드 (상위 3명)**
  - 최다 킬 / 최고 KDA(MVP) / 최고 승률
- 최근 내전 5경기 요약 목록

### 4-2. 소환사 검색 결과 `/summoner/[name]`

- 소환사 이름 헤더
- **종합 통계 카드**
  - 총 게임 수 / 승 / 패 / 승률
  - 평균 KDA (킬/데스/어시)
  - 평균 데미지 / 평균 골드
- **최근 게임 목록** (최대 20경기)
  - 승/패 배지, 챔피언 이미지, KDA, 데미지, 날짜
  - 클릭 시 게임 상세 페이지로 이동
- **챔피언별 통계 테이블**
  - 챔피언 이미지 + 이름, 게임 수, 승률, 평균 KDA

### 4-3. 게임 상세 페이지 `/match/[gameId]`

- 게임 메타 정보 (날짜, 모드, 소요 시간)
- 밴 챔피언 목록
- **팀별 스코어보드 테이블**
  - 승리팀 / 패배팀 분리
  - 소환사 이름, 챔피언 이미지, K/D/A, 데미지, 골드
  - 각 항목 팀 내 1위 하이라이트

### 4-4. 전체 랭킹 페이지 `/ranking`

- 탭: 킬 / 어시스트 / 데스 / 데미지 / 골드 / 승률 / KDA
- 모드 필터: 전체 / 협곡 / 칼바람
- 상위 N명 테이블 (이름, 수치, 게임 수)

### 4-5. 전적 목록 페이지 `/matches`

- 전체 내전 게임 목록 (최신순)
- 페이지네이션
- 모드 필터

---

## 5. API 설계 (Nuxt server routes)

Firestore 접근은 전부 서버 사이드에서만 처리 (Admin SDK 사용, 키 노출 없음).

```
GET /api/summoner/:name          → 소환사 통계 + 최근 게임 목록
GET /api/match/:gameId           → 게임 상세
GET /api/ranking?stat=&mode=     → 랭킹 (stat: kills|assists|deaths|damage|gold|winRate|kda)
GET /api/matches?page=&mode=     → 전체 게임 목록 (페이지네이션)
GET /api/summary                 → 메인 페이지용 요약 (최근 5경기 + 랭킹 상위 3)
```

## 6. UI/UX 방향

- **다크 테마** 기본 (LOL 분위기)
